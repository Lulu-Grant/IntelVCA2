# 快速开始

这篇文档为第一次接触 VCA2 的读者提供一条最短路径：先确认是否值得折腾，再按正确顺序推进。

## 1. 先判断是否属于目标用户

适合继续阅读的人群：

- 已经买到或准备购买 VCA2
- 已有服务器 / 工作站平台
- 接受老系统与高折腾成本
- 想做实验、研究或内容整理

不太适合的人群：

- 只想要便宜好用的转码卡
- 希望低功耗、低噪音、省心
- 不想碰老版本 Linux

如果更接近第二类情况，建议先看 [应用限制、风险、排错与 FAQ](./06-应用限制风险与FAQ.md) 再决定是否继续。

## 2. 最短阅读顺序

如果只想先抓住重点，可按这个顺序阅读：

1. [VCA2 概览与硬件结构](./01-VCA2概览与硬件结构.md)
2. [兼容性、前置条件与准备清单](./02-兼容性与准备清单.md)
3. [宿主机部署指南](./03-宿主机部署指南.md)
4. [Node 镜像、启动模式与持久化](./04-Node镜像与启动模式.md)

## 3. 动手前最关键的四件事

### 3.1 平台先对

优先用：

- 服务器平台
- 兼容性较好的工作站平台
- 强风道机箱

不要默认认为：

- 有 PCIe 插槽的普通台式机就能顺利点亮

### 3.2 系统先对

民间资料里成功率最高的路线仍然是：

- **CentOS 7.6 物理机**

### 3.3 资料先齐

至少要准备：

- 宿主机安装介质
- 驱动包
- 节点镜像
- 驱动包配套说明

建议直接配合 [官方驱动与依赖文件索引](./09-官方驱动与依赖文件索引.md) 一起整理下载清单。

### 3.4 预期先摆正

VCA2 更像：

- 企业级历史硬件实验品

而不是：

- 现代省心主力平台

## 4. 推荐动手顺序

### 第一步：确认硬件条件

检查：

- 平台是否靠谱
- 风道是否足够
- 供电是否到位
- 节点是否有内存

### 第二步：安装宿主系统

建议做法：

- 物理机安装兼容老系统

### 第三步：插卡并验证识别

至少确认：

- 系统能看到 VCA2 相关 PCIe 设备

### 第四步：安装驱动和节点管理工具

注意：

- 精确命令以配套驱动包版本为准

### 第五步：检查节点状态

目标：

- 三个节点状态可见

### 第六步：优先尝试 BlockIO

如果目标不只是临时点火，推荐：

- 优先走 BlockIO 持久化路线

### 第七步：打通网络和 SSH

确认：

- 节点可以通过宿主联网
- 能 SSH 登录节点

### 第八步：再考虑部署服务

比如：

- Docker
- Jellyfin
- 媒体转码测试

## 5. 新人最容易踩的坑

- 先买卡，后找资料
- 试图直接走 PVE / VFIO 直通
- 用普通台式机弱风道硬上
- 假定现代系统能直接兼容
- 在没搞清 RAMDisk / BlockIO 差别前就长期部署服务

## 6. 一条更稳的策略

如果目标是长期保留成果，建议：

- 宿主机只负责 VCA2
- 节点分工明确
- 尽量把环境持久化到 BlockIO
- 记录驱动版本、镜像来源、节点用途

## 8. 最短总结

VCA2 的最短正确打开方式是：

> 先确认平台与老系统兼容，再把宿主机和节点关系理顺，最后才去折腾服务部署。

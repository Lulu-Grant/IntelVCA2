# 兼容性、前置条件与准备清单

## 1. 先说最重要的结论

想把 VCA2 跑起来，难点通常不是某条命令，而是以下三件事同时满足：

- 平台兼容
- 系统版本对
- 驱动与镜像资料齐全

只要这三件里有一件不满足，往往就会停在“设备识别不完整”“节点打不开”或者“驱动看似装上但用不了”。

## 2. 主板和平台兼容性

现有资料对这一点非常一致：**普通家用主板大概率不适合 VCA2**。

主要原因：

- 这张卡对 **MMIO / 内存映射资源** 要求高
- 它更偏向服务器平台的资源布局
- 家用主板即使有插槽，也未必有足够资源把整张卡完整映射出来

更稳妥的平台特征：

- 服务器主板
- 兼容性好的工作站平台
- 强风道机箱
- 足够的 PCIe 与供电余量

资料中反复提到的参考方向：

- C622 一类服务器平台
- 带强风道的服务器环境

## 3. 散热要求不是小问题

VCA2 是纯被动散热设计，而内部却是三节点结构，这意味着：

- 热量并不低
- 风量不够时很容易出稳定性问题
- 普通家用机箱弱风道不适合长期跑高负载

更靠谱的散热思路：

- 服务器机箱原生强风道
- 自己改主动散热，如涡轮风扇或定向风道

不建议：

- 裸卡长期放在普通台式机箱里闷烧

## 4. 为什么 PVE / VFIO / 直通基本走不通

资料里最明确的坑之一就是这个。

原因并不是“驱动不会装”，而是底层机制和普通显卡不同：

- VCA2 使用 **NTB（Non-Transparent Bridge）**
- 底层对物理地址与节点映射关系更敏感
- 虚拟化层很难完整复刻这种硬件关系

常见表现是：

- 虚拟机里似乎能看到设备
- 驱动似乎也装上了
- 但真正到“打开节点”那一步失败

因此，如果部署方案是：

1. 现代宿主机
2. 虚拟机
3. 把 VCA2 直通进去

那大概率会卡死在最后一步。

## 5. 操作系统要求

现有资料反复指向一个现实：

- VCA2 依赖的是比较老的 Linux 生态
- 通常需要老版本 CentOS 或 Ubuntu
- 还可能需要驱动包中附带的特定内核 / 组件 / 工具

资料中最常被提到的参考方案是：

- **CentOS 7.6 物理机**

这不是因为它先进，而是因为：

- 它更接近 VCA2 原始可用生态
- 民间资料里成功案例更多

## 6. 需要准备哪些资料

准备实际部署前，建议至少备齐以下几类文件：

### 6.1 宿主机系统

- 老版本 CentOS 安装镜像
- 必要时也可对照旧版 Ubuntu 路线

### 6.2 驱动与管理工具

- 宿主机驱动包
- 节点管理工具
- 可能的内核模块、安装脚本、辅助工具

### 6.3 Node 镜像

- 官方或民间流传的节点系统镜像
- 用于 RAMDisk 或 BlockIO 的启动镜像

### 6.4 说明资料

- 驱动包内文档
- 节点管理说明
- 镜像格式和挂载说明

## 7. 硬件准备清单

上机前建议核对以下项目：

- 宿主服务器 / 工作站平台
- 合适的 PCIe 插槽
- 8Pin + 6Pin 供电
- 节点使用的 DDR4 SO-DIMM 内存
- 宿主存放镜像的硬盘空间
- 可选的 2230 / 2242 M.2 固态
- 可持续工作的强散热方案

## 8. 心理预期也要准备

VCA2 不是那种“教程复制一下就能整晚搞定”的设备。更真实的预期是：

- 点亮难度高于普通显卡
- 资料搜集本身就是门槛
- 系统必须回到旧时代
- 许多现代经验在这里不一定适用

先建立这一预期，有助于降低后续部署阶段的试错成本。

## 9. 开工前自检

正式动手前，建议先完成以下自检：

1. 是否已有合适的服务器或工作站平台？
2. 是否能提供稳定供电与强风道环境？
3. 是否已准备老系统安装介质与对应驱动？
4. 是否接受这张卡通常不能直接走现代虚拟化方案？
5. 当前目标是研究折腾，还是追求省心实用？

如果前 4 项中有 2 项以上尚未明确，通常更适合先补齐资料，再进入点亮阶段。

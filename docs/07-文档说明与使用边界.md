# 文档说明与使用边界

## 1. 这套文档是给谁看的

这套文档面向的是两类人：

- 第一次接触 VCA2，想快速判断值不值得折腾的人
- 已经拿到完整官方驱动与镜像资料，想先理清整体思路再动手的人

它的目标不是替代资料包本身，而是把分散信息重新整理成更适合阅读和理解的结构。

## 2. 这套文档解决什么问题

它主要解决这些问题：

- VCA2 到底是什么，为什么难用
- 需要什么平台和前置条件
- 宿主机和节点分别扮演什么角色
- RAMDisk 和 BlockIO 怎么选
- 节点怎么联网、怎么 SSH、怎么部署服务
- 哪些场景值得玩，哪些不值得

## 3. 哪些内容仍应以你手头官方资料包为准

虽然本仓库已经尽量整理成可读文档，但以下内容仍应优先以你手里的完整官方资料包为准：

- 驱动安装命令
- 节点管理命令
- 镜像挂载命令
- 默认账号与初始密码
- 工具目录、脚本名称、服务名
- 版本对应关系

原因不是来源不可靠，而是：

- 官方资料包内部可能存在版本差异
- 某些命令和路径会随资料包版本变化

## 4. 为什么仓库只呈现整理后的内容

这个仓库对外的定位是：

- 提供清晰、结构化、可读的使用文档

而不是：

- 把所有来源材料原样展示给最终读者

这样做的好处是：

- 首页更干净
- 阅读路径更明确
- 后来者不需要先在原始资料里自己拼装结构

## 5. 推荐使用方式

比较合适的阅读和动手方式是：

1. 先看 [快速开始](./08-快速开始.md)
2. 再按文档导航顺序读一遍
3. 真正执行安装和启动时，再对照你手头官方资料包

这样效率通常比直接翻资料包更高。

## 6. 这套文档的定位一句话总结

> 这是一套基于现有资料重写后的使用说明书，用来帮你理解和落地；而你手头的完整官方资料包负责提供最终执行细节。
